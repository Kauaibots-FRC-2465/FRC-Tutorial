# Git and Gradle Cheat Sheet

## Git

- `git branch`: list local branches
- `git switch lesson-07-arrays-lists-and-maps`: move to a lesson branch
- `git status`: see changed files
- `git diff`: inspect unstaged changes
- `git add <file>`: stage a file
- `git commit -m "Describe the change"`: save a commit

## Gradle

- `.\gradlew.bat compileJava`: compile the main Java code
- `.\gradlew.bat test`: run unit tests
- `.\gradlew.bat build`: run the standard build pipeline
- `.\gradlew.bat simulateJava`: start desktop simulation

## Good Habits

- Run `git status` before switching branches.
- Read the first real error in Gradle output.
- Fix one issue at a time.
- Keep commits small enough that you can explain them.
