# Student Lesson 15: Git for Working Programmers

This lesson explains how Git helps students work without losing progress.

## Main Ideas

Git records project history. In this course, each lesson branch is a stable snapshot of the project at one stage of learning. That means students can move between lessons without manually undoing changes line by line.

The VS Code Source Control view shows changed files, staged files, and commit messages. Git Graph shows branches and commit history. Together, these tools help students answer practical questions: what changed, what is ready to save, what branch am I on, and what happened earlier?

A commit is a named snapshot of work. Good commits are small enough that the student can explain them. If a commit message only says "stuff" or "fix," it does not teach much and it does not help later.

The important beginner habit is to check the current state before switching branches. If files are changed, the student should decide whether those changes should be committed first.

## What Students Should Understand

- Git is a history system, not just a place where code lives.
- Branches are saved lines of history.
- The Source Control view and Git Graph are enough for the course workflow.
- Small, understandable commits make future work safer.
