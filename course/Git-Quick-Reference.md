# Git Quick Reference

Git records the history of the codebase. In this course, each lesson lives on its own branch. A branch is a saved line of project history, so switching branches lets you move to a different lesson snapshot.

Students should use the VS Code Source Control view in the activity bar and the Git Graph extension. The lesson materials should not require Git commands in the terminal.

## VS Code Git Tools Students Should Know

- Source Control view: the Git view in the VS Code activity bar
- Git Graph: the branch and history viewer used in this course
- Branch picker: the VS Code control used to switch to a different lesson branch
- Changes list: the files you have edited
- Staged Changes: files marked for the next commit
- Commit message box: where you write the message for the saved snapshot you are making

## Safe Habits

- Look at the Source Control view before you switch branches.
- Commit your work before changing lessons.
- Read the branch name carefully before making changes.
- Keep commits small enough that you can explain them.
